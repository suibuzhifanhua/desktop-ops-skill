## 问题描述

电脑实际上有网络连接，但系统却显示"无法连接网络"。

---

## 解决方案

这是一个通过修改 Windows 注册表来修复该问题的方法。

### 操作步骤

1. **新建文本文档** — 在电脑上新建一个 `.txt` 文本文件。

2. **复制粘贴以下注册表代码：**

```reg
Windows Registry Editor Version 5.00

[HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Services\NlaSvc\Parameters\Internet]
"ActiveDnsProbeContent"="131.107.255.255"
"ActiveDnsProbeContentV6"="fd3e:4f5a:5b81::1"
"ActiveDnsProbeHost"="dns.msftncsi.com"
"ActiveDnsProbeHostV6"="dns.msftncsi.com"
"ActiveWebProbeContent"="Microsoft NCSI"
"ActiveWebProbeContentV6"="Microsoft NCSI"
"ActiveWebProbeHost"="www.msftncsi.com"
"ActiveWebProbeHostV6"="ipv6.msftncsi.com"
"ActiveWebProbePath"="ncsi.txt"
"ActiveWebProbePathV6"="ncsi.txt"
"CaptivePortalTimer"=dword:00000000
"CaptivePortalTimerBackOffIncrementsInSeconds"=dword:00000005
"CaptivePortalTimerMaxInSeconds"=dword:0000001e
"EnableActiveProbing"=dword:00000001
"PassivePollPeriod"=dword:0000000f
"StaleThreshold"=dword:0000001e
"WebTimeout"=dword:00000023

[HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Services\NlaSvc\Parameters\Internet\ManualProxies]
```

3. **保存为 `.reg` 文件** — 将文件重命名为 `1.reg`（注意确保文件后缀从 `.txt` 变更为 `.reg`）。

4. **双击运行** — 双击该 `.reg` 文件，按提示确认将其导入注册表。

5. **重启电脑** — 重启后问题即可解决。

---

## 原理说明

该方案修改的是 Windows **NLA（Network Location Awareness）服务**下的网络连接状态检测（NCSI）相关注册表项。Windows 通过访问微软的检测服务器（`www.msftncsi.com` 等）来判断网络是否连通，当这些检测地址被干扰或配置异常时，即使实际有网，系统也会误判为"无网络连接"。此注册表脚本将 NCSI 参数恢复为微软默认值，从而修复该问题。
